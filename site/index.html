Capabilities

The entire baking pipeline, done properly.

Getting an asset game-ready is a dozen careful steps in a strict order. Auto Bake runs all of them with the discipline of a tool built for production work.

01

Shared texture atlases

Every object lands in one atlas per map. Island scale is averaged first so texel density stays uniform, and the atlas grows up to 8192 as object counts rise.

02

Exact channel captures

Data channels bake through a temporary emission rig at one sample. Noise free, fast, and metallic surfaces keep their albedo instead of baking black.

03

UVs handled for you

Objects without UVs get a Smart UV Project automatically. Baking happens on a dedicated atlas layer, so the UVs you made by hand are never touched.

04

Materials rebuilt, originals kept

A clean Principled material is wired from the finished maps: normal node, alpha transparency, optional AO multiply. Replaced originals stay in the file.

05

Export and archive

Ship .blend, FBX, OBJ and glTF in any combination, sorted into textures and models folders, optionally packaged as a single ZIP with a README.

06

Fails like good software

Readable errors with a suggested fix, GPU to CPU fallback, a session log you can save, and every temporary change restored even when a bake dies.

STEP 1

Download the ZIP

Grab auto_bake.zip above and leave it zipped.

STEP 2

Install it

Edit › Preferences › Add-ons › Install, then pick the ZIP.

STEP 3

Enable it

Tick the checkbox next to Auto Bake in the add-on list.

STEP 4

Bake

Select objects, press N, open the Auto Bake tab, hit Bake.

Questions

Short answers to fair questions.

Is it really free?
Yes. Auto Bake is licensed GPL-3.0-or-later and the source is public. Anything you bake with it belongs entirely to you, including commercial work.
Will it work with my Blender version?
Auto Bake supports Blender 3.6 LTS through the latest stable release. Each release is verified by an automated suite that installs the add-on and runs a full bake inside real Blender 3.6, 4.2 and 5.x builds.
Does it change my scene?
Baking happens on a dedicated atlas UV layer and through temporary nodes that are removed afterwards. If you choose to replace materials, the originals are kept in the file with a fake user, so nothing is lost.
Where do my textures end up?
In the export folder you choose, sorted into textures and models subfolders, named like AutoBake_BaseColor.png. Enable ZIP export and everything is also packaged into a single archive.
GPU or CPU?
Either. Auto Bake uses your GPU when one is available and falls back to CPU cleanly when it is not. Channel passes render at one sample, so even CPU bakes stay quick.
